# ROADMAP — qtcloud-course

量潮课程云产品级路线图。覆盖三个组件（Provider / Studio / CLI）及其集成。

## 版本策略

- 组件独立版本号、独立 CHANGELOG、独立发布
- 产品级里程碑（如 v0.0.5）按产品能力组织，不按技术组件
- 当前所有组件处于 `0.MINOR.PATCH` 阶段，API 不稳定

| Scope | 版本 | 技术栈 | 状态 |
|-------|------|--------|------|
| provider | v0.0.2 | Go | ✅ REST API 完成，内存存储 |
| studio | v0.0.4 | Flutter | ✅ Sidebar 布局、PreviewScreen 场景播放 |
| cli | v0.1.0-alpha.3 | Rust | 🧪 AI 蓝图生成，功能单一 |

---

## 里程碑总览

每个里程碑由**设计意图**驱动，而非技术组件：

| 里程碑 | 设计原则 | 核心能力 | Studio | Provider | CLI |
|--------|---------|---------|--------|----------|-----|
| v0.0.5 | 原则 2+3+7 | 双向编辑 | CRUD + 拖拽排序 | CRUD API | — |
| v0.0.6 | 原则 4+6 | 资源独立+安全 | 软删除、状态校验 | 持久化 | — |
| v0.0.7 | 原则 1+5 | 双轨互通 | 蓝图导入/导出 | — | 蓝图兼容 |

---

## v0.0.5 — 双向编辑

**设计意图**：树形结构作为通用语言（原则 2）+ 支持双向工作流（原则 3）+ 操作即排序（原则 7）

编辑器不再只是查看器，真正支持增删改排。

### Studio

- [ ] 四级（Program/Course/Phase/Lesson）CRUD：新建、编辑、删除
- [ ] 发布/下架操作，发布检查子级状态
- [ ] 同级节点拖拽排序（Course / Phase / Lesson）
- [ ] 删除确认对话框，提示子级数量，已发布不可删除

### Provider

- [ ] CRUD API 对齐 Studio 编辑操作
- [ ] 排序字段支持 PATCH

---

## v0.0.6 — 资源独立 + 安全护栏

**设计意图**：资源独立性（原则 4）+ 渐进式发布与安全护栏（原则 6）

内容真正成为独立资产，不怕误操作。

### Studio

- [ ] 软删除：`deletedAt` 标记，内容不丢失可恢复
- [ ] 跨容器拖拽：Lesson 拖到其他 Phase
- [ ] 独立课时池 + "归入阶段"操作（自下而上路径）
- [ ] 编辑未保存标记
- [ ] 删除/发布全路径安全校验

### Provider

- [ ] SQLite 持久化
- [ ] 软删除 API（标记而非物理删除）
- [ ] 课时独立端点（不依赖父级路径）

### CLI

- [ ] CLI 输出 JSON 与 Provider 种子格式对齐

---

## v0.0.7 — 双轨互通

**设计意图**：双轨制生产（原则 1）+ 蓝图作为互通凭证（原则 5）

AI 生产的内容能流入 Studio，Studio 调整的内容能导出。

### Studio

- [ ] 文件导入：选择本地 JSON 文件，解析后合并到课程树
- [ ] 文件导出：当前课程树导出为 JSON
- [ ] 冲突处理策略（覆盖/跳过/保留两者）

### CLI

- [ ] 脱离 alpha
- [ ] `blueprint` 输出与 Studio 导入格式完全兼容
- [ ] 批量操作：批量创建课程/课时

### E2E

- [ ] CLI → Studio 导入全链路验证
- [ ] Studio → CLI 导出→重新导入闭环
- [ ] 冲突策略自动化测试

```
                          ▼

          │ HTTP JSON
          ▼

```

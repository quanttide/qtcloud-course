# 仪表盘

## 设计原则

### 角色分流

同一页面根据用户角色呈现不同的核心信息。课程研发人员看到研发进度和待办，教学管理人员看到班级状态和教学任务。

### 信息分层

重要指标优先展示，次要信息通过卡片折叠或二级入口访问。页面不堆砌指标，每个区域只展示一个核心数值。

### 操作直达

常用操作（新建课程、开课）在仪表盘可直接触发，不需要先进入功能页面。

## 核心元素

### 导航栏

- 左侧：产品 Logo，点击回到仪表盘
- 中间：一级导航链接——课程研发、教学管理。当前选中高亮，点击切换到对应功能页面
- 右侧：全局搜索框，输入关键词后下拉展示匹配的专业、课程、课时、班级。右侧用户头像，点击展开个人菜单

### 指标卡片

- 六张卡片分两行排列。第一行展示研发侧指标（专业数、课程数、课时数），第二行展示教学侧指标（进行中班级、学员数、今日教学待处理）
- 每张卡片显示指标数值、趋势标签（本月增减）、状态标记
- 点击卡片跳转到对应列表页并自动筛选

### 待办事项

- 按用户角色过滤。研发人员看到与内容审核相关的待办，教学人员看到与开课相关的待办
- 每项待办显示类型标签（审核/确认/检查）、标题、截止时间
- 点击复选框快速标记完成，点击标题跳转到详情页处理

### 最近更新

- 展示最近修改的专业/课程/课时/班级，每条显示名称、版本号、相对时间
- 点击跳转到对应详情页
- 点击查看全部展开完整列表

### 快捷操作

- 固定在页面底部。包含新建专业、新建课程、新建班级三个按钮
- 点击弹出创建向导，不离开当前页面

## 交互行为

- 指标卡片点击跳转后，目标页面自动聚焦到对应数据范围
- 搜索框支持拼音模糊匹配，搜索结果按类型分组展示，支持键盘上下选择
- 待办事项的复选框点击后立即标记完成（乐观更新），操作失败时恢复并提示
- 快捷操作的创建向导采用步骤式模态框，每一步可回退

## 交互流程

### 查看研发概览

用户进入系统 → 仪表盘展示六张指标卡片 → 查看第一行研发指标 → 点击"课程数"卡片 → 跳转到课程列表页，自动筛选当前用户负责的课程 → 浏览课程详情

### 处理待办事项

用户登录 → 查看待办列表 → 点击复选框标记完成 → 系统标记成功并更新待办计数 → 点击待办标题 → 跳转到对应处理页面 → 完成处理后返回仪表盘 → 待办项消失

### 快速创建课程

用户在仪表盘 → 点击"+ 新建课程" → 弹出创建向导（第一步选专业，第二步填课程基本信息，第三步设置课时结构） → 完成后自动跳转到课程详情页 → 返回仪表盘后课程数卡片更新

# ROADMAP — Studio

## 当前目标：v0.0.5 课程编辑可用

### Phase 1（P0）：四级 CRUD + 拖拽排序

**现状**：课程编排树只读，用户能浏览六级结构但无法创建、编辑、删除节点。

**目标**：实现课程结构的 CRUD + 排序，让用户能通过 Studio 完成课程编排。

- [ ] Program / Course / Phase / Lesson 四级新建对话框
- [ ] 四级编辑（名称/描述可修改）
- [ ] 四级删除确认对话框，提示子级数量
- [ ] 同级节点拖拽排序
- [ ] 发布/下架操作
- [ ] 发布 Course 时检查 Lesson 状态，有草稿则提示
- [ ] 已发布节点不可删除

### Phase 2（P1）：资源独立 + 安全护栏

- [ ] 软删除（deletedAt 标记，内容可恢复）
- [ ] 跨容器拖拽（Lesson 拖到其他 Phase）
- [ ] 独立课时池 + "归入阶段"操作（自下而上路径）
- [ ] 编辑未保存标记

### Phase 3（P2）：双轨互通

- [ ] 文件导入（选择本地 JSON）
- [ ] 文件导出（当前树导出为 JSON）
- [ ] 冲突处理策略（覆盖/跳过/保留两者）

### 交付标准

```
flutter test                          # ✅ 全部通过
dart analyze                          # ✅ 零报错
# 验收：纯 Studio 操作路径
flutter run -d linux                   # 创建专业→添加课程→新建课时→发布→删除→恢复
```

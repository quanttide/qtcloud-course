# 工作流：从蓝图到落地

## 概述

课程制作分两条路径，最终在 Provider 数据层汇合：

```
┌──────────────────────┐     ┌──────────────────────┐
│        CLI           │     │       Studio         │
│   qtcloud-course     │     │   Flutter 桌面端       │
│   面向 AI            │     │   面向人类             │
│                      │     │                      │
│  blueprint 生成     │     │  课程编排树编辑        │
│  → 批量产出课程JSON  │     │  → 手动创建/调整      │
│  → 适用于大规模导入   │     │  → 适用于精细微调     │
└────────┬─────────────┘     └──────────┬───────────┘
         │                              │
         ▼                              ▼
    ┌────────────────────────────────────────┐
    │              Provider (Go)              │
    │   REST API :8080                        │
    │   种子数据导入 / 运行时 CRUD             │
    └────────────────────────────────────────┘
```

- **CLI 路径**：AI 生成课程蓝图 JSON → Provider 种子导入 → Studio 可浏览/调整/发布
- **Studio 路径**：人类在图形界面中直接编排 → Provider API 读写

两条路径产出的数据格式一致，可互操作。

## 课程编排树

Studio 课程研发板块的核心是 **Program → Course → Phase → Lesson → Scene → Step** 六级树：

| 层级 | 含义 |
|------|------|
| Program | 专业，顶层教学计划 |
| Course | 课程，教学单元 |
| Phase | 阶段，课程内部分期 |
| Lesson | 课时，最小内容组织单元 |
| Scene | 场景，课时的分支视频片段 |
| Step | 步骤，场景内的操作引导 |

### 当前 Studio 已具备的能力

| 操作 | 状态 | 说明 |
|------|------|------|
| 浏览六级树 | ✅ | 展开/折叠，只读展示 |
| 查看状态标记 | ✅ | StatusChip 显示草稿/已发布 |
| 试听预览课时 | ✅ | PreviewScreen 逐场景逐步骤引导 |

## 路径 A：AI 先导（CLI → Provider → Studio）

### 适用场景

新专业批量创建、课程体系重构、从旧系统迁移。AI 完成课程树设计，人类在 Studio 中审阅和微调。

### 完整流程

```
1. 需求分析 (人类)
   ↓
2. CLI blueprint 生成课程大纲 (AI)
   ↓
3. 大纲 JSON 导入 Provider 种子数据
   ↓
4. Studio 启动，课程树展示完整结构
   ↓
5. 人类在 Studio 中审阅 → 调整 → 发布
```

### 用户在 Studio 中的操作序列

```
进入课程研发页 → 课程树已展示蓝图导入的完整结构
  │
  ├─ 浏览各层级，检查结构是否合理
  │   ├─ 展开 Program 查看 Course 列表
  │   ├─ 展开 Course 查看 Phase 列表
  │   └─ 展开 Phase 查看 Lesson 列表
  │
  ├─ 发现名称或描述不合适
  │   └─ 点击节点 → 进入编辑态 → 修改 → 保存
  │
  ├─ 发现缺少阶段或课时
  │   └─ 点击所在层级列表末尾的「+ 新建」
  │      → 填写名称
  │      → 确认，节点插入树中（状态默认为草稿）
  │
  ├─ 发现顺序不对
  │   └─ 长按节点拖拽到目标位置
  │
  ├─ 需要删除多余节点
  │   └─ 右键节点 → 「删除」
  │      → 弹出确认框（提示包含的子级数量）
  │      → 确认后节点从树中移除
  │
  ├─ 准备发布
  │   └─ 右键节点 → 「发布」
  │      → 发布 Course 时检查下属 Lesson 状态
  │        如果有草稿课时 → 提示确认
  │      → 节点状态标记更新为"已发布"
  │
  └─ 所有节点发布完成 → 课程可被教学管理引用
```

### 涉及的功能

| 操作 | 级别 | 边界定义 |
|------|------|----------|
| 编辑名称/描述 | P0 | 仅 Program、Course、Phase、Lesson 四级的名称和描述字段。点击进入编辑态，底部 [保存] [取消] |
| 新建层级节点 | P0 | Program、Course、Phase、Lesson 四级。新建的节点 status=草稿，sortOrder=同级最大值+1。Scene 和 Step 不单独新建 |
| 删除层级节点 | P0 | 弹出确认提示子级数量。删除 Program→下属节点不再展示，但不删除独立资源。不可删除已发布节点 |
| 拖拽排序 | P1 | 同级节点之间重新排序，不可跨级。拖拽时显示插入指示线 |
| 发布/下架 | P1 | Program、Course、Lesson 三级可发布。Phase 不独立发布。发布 Course 时有草稿课时则提示确认 |

## 路径 B：Studio 原生（人类直接操作）

### 适用场景

单个课程创建、已有课程微调、课时内容编辑。人类从零开始在界面中搭建课程树。

### 完整流程

```
1. 新建 Program
   ↓
2. 逐级新建 Course → Phase → Lesson
   ↓
3. 填充 Lesson 中的 Scene → Step 内容
   ↓
4. 试听预览验证
   ↓
5. 发布
```

### 用户在 Studio 中的操作序列

```
进入课程研发页 → 课程树为空（或已有其他专业）
  │
  ├─ 点击「+ 新建专业」
  │   → 填写专业名称、描述
  │   → 确认，树中出现新专业节点
  │
  ├─ 展开专业，点击「+ 新建课程」
  │   → 填写课程名称、描述
  │   → 确认，专业下出现新课程节点
  │
  ├─ 展开课程，点击「+ 新建阶段」
  │   → 填写阶段名称
  │   → 确认，课程下出现新阶段节点
  │   → 可继续新建多个阶段，拖拽调整顺序
  │
  ├─ 展开阶段，点击「+ 新建课时」
  │   → 填写课时标题、时长
  │   → 确认，阶段下出现新课时节点
  │   → 可继续新建多个课时，拖拽调整顺序
  │
  ├─ 展开课时，进入内容编辑
  │   └─ (暂未实现：Scene→Step 的编排界面)
  │
  ├─ 点击课时旁的试听按钮 → 预览效果
  │
  ├─ 确认无误后逐个发布
  │
  └─ 所有节点发布完成 → 课程可被教学管理引用
```

### 涉及的功能

与路径 A 相同，只是入口不同——路径 A 的入口是"审阅导入的蓝图"，路径 B 的入口是"从空白新建"。

## 路径 C：混合模式（CLI 生成骨架 → Studio 精调）

### 适用场景

日常最常用的流程。AI 负责课程结构设计（骨架），人类负责内容填充（血肉）。

### 完整流程

```
1. 人类描述课程需求
   ↓
2. AI 通过 CLI 生成课程骨架 JSON
   （Program→Course→Phase→Lesson，不含 Scene→Step 内容）
   ↓
3. 骨架 JSON 导入 Studio
   ↓
4. 人类在 Studio 中检查和调整
   ├─ 修改名称/描述/排序
   ├─ 增删阶段和课时
   └─ 逐个进入课时，填充 Scene→Step 内容
   ↓
5. 预览验证
   ↓
6. 发布
```

### 用户在 Studio 中的操作序列

```
进入课程研发页
  │
  ├─ 点击「导入蓝图」
  │   → 选择 JSON 文件或粘贴内容
  │   → 导入的课程树插入现有列表
  │   → 如果有同名节点 → 提示「覆盖 / 跳过 / 保留两者」
  │
  ├─ 浏览骨架结构（同路径 A）
  │
  ├─ 调整结构（编辑名称、新建、删除、排序）
  │
  ├─ 逐个进入课时填充内容（暂未实现）
  │
  ├─ 预览验证
  │
  └─ 发布
```

### 涉及的功能

| 操作 | 级别 | 边界定义 |
|------|------|----------|
| 导入蓝图 | P2 | 选择 JSON 文件或粘贴，解析后自动分配 ID，同名冲突提示处理方式 |
| 导出蓝图 | P2 | 选中节点及其子级导出为 JSON，可被 CLI 或其他 Studio 实例使用 |
| 编辑/新建/删除/排序/发布 | P0/P1 | 与路径 A 定义一致 |

## 蓝图规格

CLI `blueprint` 命令产出的 JSON 与 Studio 数据模型完全兼容。

### 最小示例

      "courses": [
        {
          "id": "course-1",
          "name": "Python 基础",
          "description": "Python 基础语法与工具链",

### CLI 到 Studio 的导入流程

```
CLI blueprint 生成 → seed.json → Provider 种子数据
                                    ↓
                              Studio 启动
                                    ↓
                          assets 模式或 API 模式
                                    ↓
                          课程树展示在 ProgramScreen
```

## 数据一致性规则

- CLI 产出的 JSON 可直接作为 Provider 种子数据，Studio 无需二次转换
- Studio 编辑后的数据可通过 Provider API 导出，与 CLI 蓝图格式互通
- 所有层级节点的 status 统一使用 `draft` / `published` 俩状态
- 层级间通过嵌套结构（而非独立 ID 引用）组织，一个 JSON 文件包含完整树

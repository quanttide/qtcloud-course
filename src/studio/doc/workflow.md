# 工作流：从蓝图到落地

## 概述

课程制作分两条路径，最终在 Provider 数据层汇合：

```
┌──────────────────────┐     ┌──────────────────────┐
│        CLI           │     │       Studio         │
│   qtcloud-course     │     │   Flutter 桌面端       │
│   面向 AI            │     │   面向人类             │
│                      │     │                      │
│  blueprint 生成     │     │  课程编排树编辑        │
│  → 批量产出课程JSON  │     │  → 手动创建/调整      │
│  → 适用于大规模导入   │     │  → 适用于精细微调     │
└────────┬─────────────┘     └──────────┬───────────┘
         │                              │
         ▼                              ▼
    ┌────────────────────────────────────────┐
    │              Provider (Go)              │
    │   REST API :8080                        │
    │   种子数据导入 / 运行时 CRUD             │
    └────────────────────────────────────────┘
```

- **CLI 路径**：AI 生成课程蓝图 JSON → Provider 种子导入 → Studio 可浏览/调整/发布
- **Studio 路径**：人类在图形界面中直接编排 → Provider API 读写

两条路径产出的数据格式一致，可互操作。

## 课程编排树

Studio 课程研发板块的核心是 **Program → Course → Phase → Lesson → Scene → Step** 六级树：

| 层级 | 含义 |
|------|------|
| Program | 专业，顶层教学计划 |
| Course | 课程，教学单元 |
| Phase | 阶段，课程内部分期 |
| Lesson | 课时，最小内容组织单元 |
| Scene | 场景，课时的分支视频片段 |
| Step | 步骤，场景内的操作引导 |

## 要做什么？

### 场景 1：从零开始搭建一门新课

适用：讲师要开一门新课时，课程树上还没有任何相关内容。

```
进入课程研发页
  │
  ├─ 在列表底部点击「+ 新建 Program」
  │   → 填写名称、描述
  │   → 确认 → 新 Program 出现在列表中（状态：草稿）
  │
  ├─ 展开 Program → 点击「+ 新建 Course」
  │   → 填写名称、描述
  │   → 确认 → 新 Course 出现在 Program 下
  │
  ├─ 展开 Course → 点击「+ 新建 Phase」
  │   → 填写名称
  │   → 确认 → 新 Phase 出现在 Course 下
  │   → 可继续新建多个 Phase
  │
  ├─ 展开 Phase → 点击「+ 新建 Lesson」
  │   → 填写标题、时长
  │   → 确认 → 新 Lesson 出现在 Phase 下
  │   → 可继续新建多个 Lesson
  │
  └─ 新建完成后，可对任意节点进行编辑、排序、删除、发布
```

### 场景 2：从 CLI 蓝图导入后审阅调整

适用：AI 通过 CLI 批量生成了课程大纲，人类需要检查并微调。

```
进入课程研发页
  │
  ├─ 点击「导入蓝图」
  │   → 选择 JSON 文件或粘贴内容
  │   → 系统解析后插入课程列表
  │   → 如果存在同名 Program → 提示：

        ╔══════════════════════════════════════╗
        ║  同名专业「AI 编程入门」已存在       ║
        ║                                      ║
        ║  ○ 覆盖（替换原有内容）              ║
        ║  ○ 跳过（保留原有，不导入此项）      ║
        ║  ○ 保留两者（导入为新的）            ║
        ║                                      ║
        ║        [确认]    [取消]              ║
        ╚══════════════════════════════════════╝
  │
  ├─ 浏览导入的课程树结构
  │   ├─ 展开 Program 查看 Course 列表
  │   ├─ 展开 Course 查看 Phase 列表
  │   └─ 展开 Phase 查看 Lesson 列表
  │
  ├─ 需要修改 → 找到目标节点
  │   ├─ 修改名称或描述 → 点击节点 → 进入编辑态 → 修改 → [保存]
  │   ├─ 增加内容 → 在对应层级点击「+ 新建」→ 填写 → 确认
  │   ├─ 删除内容 → 右键节点 → [删除] → 确认弹窗 → 移除
  │   └─ 调整顺序 → 长按节点 → 拖拽到目标位置
  │
  └─ 确认无误 → 选择要发布的节点 → [发布]
```

### 场景 3：在已有课程结构中做局部调整

适用：课程已经在用了，需要加一个新课时、调整阶段顺序、修改某门课的名称。

```
进入课程研发页
  │
  ├─ 找到要调整的目标节点（浏览展开或搜索）
  │
  ├─ 目标：加一个新课时到某阶段下
  │   └─ 展开到目标 Phase → 点击「+ 新建 Lesson」
  │
  ├─ 目标：修改某门课的名称
  │   └─ 找到目标 Course → 点击名称 → 进入编辑态 → 修改 → [保存]
  │
  ├─ 目标：调整阶段顺序
  │   └─ 找到目标 Course → 展开 → 长按 Phase → 拖拽到正确位置
  │
  ├─ 目标：删除一个多余的课时
  │   └─ 找到目标 Lesson → 右键 → [删除] → 确认
  │
  └─ 操作完成，其他节点不受影响
```

### 场景 4：批量发布/下架

适用：课程结构已确定，需要将多个节点从草稿切换为已发布，或将已发布的课程下架。

```
进入课程研发页
  │
  ├─ 找到要发布的节点
  │
  ├─ 发布一个 Course
  │   └─ 右键 Course → [发布]
  │      → 如果该 Course 下有草稿 Lesson → 弹窗提示：

        ╔══════════════════════════════════════╗
        ║  Course「数据工程」包含 2 个草稿课时 ║
        ║                                      ║
        ║  仍要发布吗？草稿课时对教学不可见。   ║
        ║                                      ║
        ║        [确认发布]  [取消]            ║
        ╚══════════════════════════════════════╝

  │
  ├─ 发布一个 Program
  │   └─ 右键 Program → [发布] → 状态更新
  │
  ├─ 下架一个已发布的节点
  │   └─ 右键已发布的节点 → [下架] → 状态切为草稿
  │
  └─ 发布/下架后，教学管理侧引用该节点的班级会看到状态变化
```

### 场景 5：导出课程数据给 CLI 或其他实例

适用：需要将 Studio 中的课程结构导出为蓝图 JSON，供 CLI 处理或分享给其他团队。

```
进入课程研发页
  │
  ├─ 找到要导出的节点（Program 或 Course）
  │
  └─ 右键节点 → [导出为蓝图]
     → 系统生成 JSON 文件并触发下载
     → JSON 格式与 CLI blueprint 输出兼容
```

## 功能边界定义

### P0 — 课程结构编辑

**新建节点**

- 范围：Program、Course、Phase、Lesson 四级。Scene 和 Step 不单独新建
- 行为：在当前层级下创建空节点，新建后名称可编辑，焦点自动落入
- 数据：status 默认为 `draft`，sortOrder 自动设为同级最大值 +1

**编辑节点**

- 范围：Program、Course、Phase、Lesson 四级的名称和描述字段
- 行为：点击节点标题进入编辑态，底部展示 [保存] [取消]
- 约束：未保存时节点显示未保存标记。不涉及 Scene/Step 的内容编辑

**删除节点**

- 范围：Program、Course、Phase、Lesson 四级
- 行为：弹出确认对话框，提示被删除项包含的子级数量
- 级联：删除 Program 后其下 Course/Phase/Lesson 不再展示，但不删除独立资源
- 约束：不可删除已发布状态的节点

### P1 — 发布与排序

**状态切换**

- 范围：Program、Course、Lesson 三级。Phase 不独立发布，随 Course 发布
- 行为：draft → published：节点状态标记更新。published → draft：节点下架
- 约束：发布 Course 时检查下属 Lesson 状态，有草稿课时则提示确认

**拖拽排序**

- 范围：同级节点之间（Course 在 Program 内、Phase 在 Course 内等）
- 行为：拖拽时显示插入位置指示线，松手后保存新顺序
- 约束：不可跨级拖拽

### P2 — 蓝图互通

**导入蓝图**

- 触发：「导入蓝图」按钮 → 文件选择器或粘贴 JSON
- 输入：与蓝图示例格式一致的 JSON
- 行为：解析后自动分配 ID，同名节点弹出冲突处理对话框

**导出蓝图**

- 触发：右键节点 → [导出为蓝图]
- 输出：选中节点及其子级导出为兼容 CLI 蓝图格式的 JSON

## 蓝图规格

CLI `blueprint` 命令产出的 JSON 与 Studio 数据模型完全兼容。

### CLI 到 Studio 的导入流程

```
CLI blueprint 生成 → seed.json → Provider 种子数据
                                    ↓
                              Studio 启动
                                    ↓
                          assets 模式或 API 模式
                                    ↓
                          课程树展示在 ProgramScreen
```

## 数据一致性规则

- CLI 产出的 JSON 可直接作为 Provider 种子数据，Studio 无需二次转换
- Studio 编辑后的数据可通过 Provider API 导出，与 CLI 蓝图格式互通
- 所有层级节点的 status 统一使用 `draft` / `published` 俩状态
- 层级间通过嵌套结构（而非独立 ID 引用）组织，一个 JSON 文件包含完整树
